# AIメタ認知フィードバックループ (Self-Correction Engine)

## 概要

対話中にユーザーの修正・無視・明示指示を検知し、AIが自律的に振る舞いを改善するスキル。
検知 → 分析 → 適用の3ステップで処理し、軽微なものは即時適用、重要なものは `preferences.md` / ``principles.md`` へ恒久記録する。

---

## トリガー条件

対話中、バックグラウンドで常時稼働。以下の**いずれか**を検知したときに発動する。

**手動発動:** ユーザーが「メタ認知チェック」「振る舞い振り返って」等と発言した場合、直近の対話を遡って未検知のパターンがないか能動的にスキャンする。

### 検知パターン一覧

| # | パターン分類 | 具体的な検知シグナル | 例 |
|---|------------|------------------|----|
| P1 | 出力形式の手動修正 | ユーザーがAIの出力を貼り直し・書き直した | 箇条書きを段落文に書き直した |
| P2 | 提案の意図的無視 | 複数回の提案を採用せず別の方向に進めた | 「それはいいです」+独自の方法を実行 |
| P3 | 明示的スタイル指示 | 「もっと〜して」「〜はやめて」等の直接指示 | 「結論だけ出して、理由は省いて」 |
| P4 | 繰り返し同じ修正 | 同セッション内で同種の修正が2回以上発生 | 毎回「箇条書きにして」と言い直す |
| P5 | 期待外れの無言反応 | 提案後に沈黙→別話題に切り替え（応答なし） | 提案を完全スルーして「次に進んで」 |
| P6 | 否定的フィードバック | 「違う」「そうじゃなくて」「ズレてる」等 | 「そういう意味じゃない」 |
| P7 | 粒度・詳細度の調整 | 「もっと詳しく」「要約して」「短くして」 | 「2行でまとめて」 |
| P8 | ツール使用の修正 | ユーザーが別のツール/ファイルを指定し直した | AIが別ファイルに書いたものを正しいパスに移した |

---

## 処理フロー（3ステップ）

### STEP 1: Detect（検知）

```
検知後、内部で以下を即時評価する：

1. どのパターン（P1〜P8）に該当するか
2. 今セッションで同じパターンが何回発生しているか（頻度カウント）
3. 誤検知の可能性はないか（後述: 誤検知防止チェック）
```

### STEP 2: Analyze（分析）

```
以下の観点で原因を特定する：

- 【出力形式の問題】 → preferences.md「出力スタイル」セクション更新候補
- 【情報量・粒度の問題】 → preferences.md「詳細度」セクション更新候補
- 【理解の齟齬】 → 要件定義不足、次回は確認質問を先行させる
- 【ドメイン知識の不足】 → 05_Knowledge参照を怠った可能性
- 【原則違反】 → `principles.md` との乖離がないか確認
```

### STEP 3: Apply（適用）

```
重要度に応じて以下のいずれかを実行する：

- Minor → 即時適用のみ（ファイル更新なし）
- Medium → 今セッション中に適用 + preferences.md に追記（報告必須）
- Major → 即時適用 + preferences.md / `principles.md` に追記 + ユーザーへ明示報告
```

---

## 重要度分類

| 重要度 | 判定基準 | 対応アクション |
|--------|---------|--------------|
| **Minor** | 初回検知、一時的な要望の可能性がある | 即時適用のみ。ファイル更新なし |
| **Medium** | 同セッション内で2回以上検知、または明示的なスタイル指示（P3） | 即時適用 + `preferences.md` に追記 |
| **Major** | セッションをまたいで繰り返される、または ``principles.md`` レベルの価値観的指摘 | 即時適用 + 両ファイル更新 + ユーザーへ明示報告 |

## 出力フォーマット

### パターン別レポートテンプレート

#### Minor（サイレント適用）
```
（報告なし。内部で修正を適用し、次の応答から反映する）
```

#### Medium（セッション内通知）
```
[メタ認知] {パターン分類} を検知しました。
- 検知内容: {何をどう修正されたか}
- 適用: {今後どう変える}
- preferences.md への記録: ✅ 完了
```

#### Major（明示報告 + ファイル更新）
```
[メタ認知・重要] {パターン分類} を検知しました。

## 検知内容
{具体的に何が起きたか}

## 原因分析
{なぜAIがミスをしたか}

## 適用内容
- 今後の振る舞い: {変更点}
- preferences.md 更新: ✅ 完了
- `principles.md` 更新: {必要なら ✅ / 不要なら「対象外」}

## 再発防止
{同じ問題を起こさないための具体的な対策}
```

### preferences.md 追記フォーマット
```
[AI-Self-Correction] {YYYY-MM-DD} ({重要度}): ユーザーは{検知内容}を好む/嫌うため、今後は{変更する振る舞い}。検知パターン: P{番号}
```

### `principles.md` 追記フォーマット（Majorのみ）
```
[AI-Self-Correction] {YYYY-MM-DD}: {原則名} — {学んだ原則の内容}（メタ認知スキルによる自動記録）
```

## 誤検知防止チェック

以下に該当する場合は**発動しない**（誤検知として無視する）：

- ユーザーが「試しに〜して」「仮に〜とすると」等の実験的文脈で別の書き方を求めた場合
- ユーザーが他者への共有用に形式を変えた（AIへのフィードバックではなく成果物フォーマット変更）
- 1回のみの指示で、以後同じ形式を受け入れている場合
- 技術的制約による変更（ファイルパスの誤り修正など、スタイルとは無関係）

**誤検知の閾値:** 同一パターンが1回のみかつ文脈が一時的な場合はMajor/Mediumに分類しない。

## 制約・注意事項

- **自動書き換え禁止:** `CLAUDE.md` 本体の書き換えはユーザー承認後のみ実施する。`preferences.md` / ``principles.md`` はMedium以上で自動追記可。
- **過剰報告の抑制:** Minorレベルの修正を毎回報告するとノイズになるため、報告はMedium以上に限定する。
- **セッションをまたぐ記憶:** `preferences.md` への記録が唯一の永続化手段。必ずフォーマット通りに追記すること。
- **後退防止:** 一度記録した preferences は、後続セッションで必ず読み込まれる前提で記述する（曖昧表現禁止）。
- **週次レビューとの連携:** `00_Self/_effectiveness.md` へのサマリ記録は週次レビュースキルの「自己省察とプロンプト改善」フェーズと連動する。
