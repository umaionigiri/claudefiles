# 大阪市統合PF 月次納品: 命名規則と編集ルール

## ベースフォルダ
`/mnt/c/Users/shoken.ohshiro/Accenture/大阪市_統合PF-XX_管理者(Mup) - 01_納品/`

## 当月納品フォルダ名
`<YYYYMMDD>_第n回目`
- `YYYYMMDD` = **納品月の月末日**（例: 4月分 → `20260430`、5月分 → `20260531`）
- `n` = 既存フォルダ末尾「N回目」の最大値 +1（「第」あり/なし両対応）

## サブフォルダ階層（すべて空フォルダ）
```
第n回中間分成果物/
  ├ プロジェクト管理/
  │   ├ リスク管理表/         (Step 5 で【統合PF】全体_リスク管理簿_<月末>.xlsx を格納)
  │   ├ 課題管理表/           (Step 5 で【統合PF】全体_課題・ToDo管理簿_<月末>.xlsx を格納)
  │   ├ 会議資料/
  │   │   ├ 作業部会/         (Step 3 で 第N回フォルダをコピー、議事録は除外)
  │   │   └ 進捗会議/
  │   └ 議事録/
  │       ├ 作業部会/         (Step 4 で v1.00 議事録 or プレースホルダを格納)
  │       └ 進捗会議/
  ├ ガバナンス検討/
  └ 本稼働前対応/
      ├ 事業者間質問管理表/
      ├ 稼働状況報告書/         (Step 6 で **前月度** <YYYY年M月度> フォルダをコピー)
      └ 本稼働前対応計画書/
```

## 当月フォルダ直下のファイル

### 1. 検査仕様書 (xlsx)
- ファイル名: `<納品月の月末日:YYYYMMDD>_検査仕様書（様式）.xlsx` （**括弧は全角**）
- 編集箇所: メインシート（先頭シート）の名前を `第n回中間分` に変更
- セル内容・色・フォント・枠線は **触らない**

### 2. 業務完了報告 (docx)
- ファイル名: `業務完了報告.docx`
- 編集箇所:
  - P1（右寄せ日付段落）: `令和X年M月D日　` （X = 月末日の令和暦年、M=月、D=日）
  - Table 0 R5 C1（履行内容セル）: `令和X年度　第n回中間分業務` （X = 4-3 会計年度の令和、間に全角スペース）
- 段落・セル内の他の run、フォント、配置は **触らない**

### 3. 統合PF請求書 (docx)
- ファイル名: `統合PF請求書(第n回中間分業務).docx` （**括弧は半角**、既存命名に合わせる）
- 編集箇所:
  - Table 0 R2 セル（26列マージ済みの右上日付セル）: `　令和X年　M月　15日`（翌月15日）
- レイアウト・フォント・他のセルは **触らない**

## 令和年度の決め方

### 令和暦年（カレンダー年）
- 令和元年 = 2019年
- 令和X年 = 西暦 - 2018

### 令和会計年度（4月始まり）
- 4月〜12月: 令和(西暦 - 2018) 年度
- 1月〜3月: 令和(西暦 - 2019) 年度
  - 例: 2026年4月分 → 令和8年度 / 2027年1月分 → 令和8年度 / 2027年4月分 → 令和9年度

## 会議資料コピー（Step 3） / 議事録抽出（Step 4）

ソース（**お客さん環境、read-only**）:

| ラベル | パス | フォルダ名パターン |
|-------|------|------------------|
| 作業部会 | `…/B2B_大阪市…/01_作業部会` | `第N回作業部会_YYYYMMDD（任意のサフィックス）` |
| 進捗会議 | `…/B2B_大阪市…/02_進捗会議` | `第N回進捗会議_YYYYMMDD（任意のサフィックス）` |

両ソースとも、納品月の年月（YYYYMM）と日付部分の前6桁が一致するフォルダが対象。例: 納品月 = 2026年3月 → 作業部会 第51〜54回 + 進捗会議 第16〜17回。

### Step 3: 会議資料/{作業部会, 進捗会議}/

各会議フォルダを対応するラベル配下にコピー。ただし **議事録ファイル（ファイル名に「議事録」を含むもの）は除外**（`shutil.copytree` の `ignore` コールバックで再帰的に弾く）。これにより会議資料配下に議事録は一切混入しない。

### Step 4: 議事録/{作業部会, 進捗会議}/

各会議フォルダの **トップレベル直下** にあるファイルから、「議事録」を含むファイルを検索：

| 状態 | 振る舞い |
|------|---------|
| `v1.00` を含む議事録ファイルが見つかった | そのファイルを `議事録/{ラベル}/第N回…_YYYYMMDD/` にコピー |
| 議事録ファイルなし（書面開催等）、または `v1.00` 以外（例: `v0.01`）のみ | 同フォルダに `※確認中のため次月納品.txt` を作成（本文に理由を記載） |

`old/` サブフォルダ内の旧バージョン議事録は**対象外**（top-level のみ参照）。

新しい会議種別を追加するには `MEETING_SOURCES` リスト（`scripts/create_delivery_folder.py`）に `{label, src, re}` を追加する。

### 顧客環境ルート（READ-ONLY 厳守）

以下の SharePoint ルートはすべて **お客さん環境**。**いかなる書き込み・更新・削除・名前変更も禁止**。許される操作は read-only のみ（`iterdir()`、`shutil.copytree(src,…)`、`shutil.copyfile(src,…)`、`Path.exists()` 等）。

| 定数 | パス | 用途 |
|-----|------|------|
| `CUSTOMER_DOC_ROOT` | `…/osakacitycommunication/B2B_大阪市統合プラットフォーム導入・構築 - ドキュメント` | 会議資料、議事録、各種管理簿、稼働状況報告書 |
| `CUSTOMER_RENRAKU_ROOT` | `…/osakacitycommunication/B2B_大阪市統合プラットフォーム連絡チーム - ドキュメント` | サービス横断会議連絡台帳 |

新しい顧客環境ルートを扱う場合も同じルールを継承する（src 引数として渡すのみ、書き込みは納品先のみ）。

## 重要な制約

- **既存の納品フォルダ・ファイル（過去の `<日付>_<n>回目` フォルダ）には絶対に書き込まない**
- **顧客環境の `01_作業部会` 配下には絶対に書き込まない**（`shutil.copytree` の src 参照のみ。書き込みはコピー先のみ）
- テンプレートは `assets/templates/` 配下のスキル同梱版のみを使う
- 統合PF請求書 .doc（旧形式）は python-docx 非対応のため、テンプレは `.docx` 化済み（既に変換完了）
- スキル実行は **スモークテスト方式**: 各ステップ完了後に `AskUserQuestion` でユーザー確認を取る
