# RAG検索精度向上プラン

## 現状の理解

### 現在のRAGアーキテクチャ
- **検索方式**: Gemini File Search Store（セマンティック検索 / ベクトル埋め込み）
- **データソース**:
  - Q&A JSON（コールセンター・ヘルプデスク履歴）
  - Web Markdown（公式サイトクローリング）
- **検索温度**: 0.1（決定論的）
- **取得件数**: 上位3件

### 課題
「参加年齢」など正確に回答したい内容について、質問時に該当ファイルが検索結果として抽出されるか不確実

---

## 質問への回答

### Q1: 想定質問を文書内に入れる方法は有効か？

**✅ 有効です**

セマンティック検索はベクトル埋め込みによる意味的類似度で検索するため、想定質問を文書に含めることで：
- ユーザー質問と文書の類似度が向上
- 該当ファイルが検索ヒットしやすくなる

**推奨フォーマット例**:
```markdown
# 参加年齢について

## よくある質問
- 何歳から参加できますか？
- 参加資格の年齢制限は？
- 30歳未満でも参加できますか？

## 回答
WMG2027への参加資格は**30歳以上**です（2027年12月31日時点）。
競技によっては追加の年齢条件がある場合があります。
```

### Q2: PDF読み取り精度向上 vs マルチモーダルRAG

**現在のシステム**: テキストベースRAG（JSON、Markdown）

| 方式 | 説明 | メリット | デメリット |
|------|------|---------|-----------|
| **テキスト抽出** | PDF→Markdown変換 | シンプル、現行システムで対応可 | 図表・レイアウト情報が失われる |
| **マルチモーダルRAG** | 画像として認識 | 図表・レイアウト理解可能 | 実装複雑、コスト増 |

**結論**:
- FAQデータは**テキストで十分**（図表より文章が重要）
- PDFを使う場合は**Markdown変換**が現実的

## 改善提案

### 提案1: 正確性が重要な項目の専用FAQファイル作成（推奨）

```
data/
└── faq/
    ├── age-requirements.md      # 参加年齢
    ├── registration-dates.md    # 登録期間
    ├── venue-information.md     # 会場情報
    └── ...
```

**ファイル構造**:
```markdown
---
category: 参加資格
keywords: [年齢, 何歳, 30歳, 参加条件]
priority: high
---

# 参加年齢について

## 想定質問
- 何歳から参加できますか？
- 参加資格の年齢制限は？
- 年齢条件を教えてください

## 公式回答
WMG2027への参加資格は**30歳以上**（2027年12月31日時点）です。
```

### 提案2: Q&Aペア形式でのデータ強化

現在のQ&A JSONを拡張し、類似質問のバリエーションを追加:

```json
{
  "question": "参加年齢は？",
  "similar_questions": [
    "何歳から参加できますか",
    "年齢制限はありますか",
    "30歳未満でも参加できますか"
  ],
  "answer": "30歳以上（2027年12月31日時点）が参加資格です。",
  "category": "参加資格",
  "confidence": "high"
}
```

### 提案3: 検索プロンプトの最適化

重要カテゴリーへの重み付けを検討:
- `src/server/lib/gemini/file-search/prompts.ts` を調整

## 実装フロー（スプレッドシート管理の場合）

### 対象カテゴリー（全て重要）
- **参加資格系**: 年齢、居住地、競技レベルなど
- **日程・スケジュール系**: 開催期間、登録期間、締切など
- **費用・支払い系**: 参加費、支払い方法、キャンセルポリシーなど

### Step 1: スプレッドシートに「想定質問」列を追加

**推奨フォーマット**:
| カテゴリ | 質問（正式） | 想定質問（バリエーション） | 回答 | 重要度 |
|---------|-------------|-------------------------|------|--------|
| 参加資格 | 参加年齢は？ | 何歳から参加できますか？/30歳未満は？/年齢制限は？ | 30歳以上（2027年末時点） | high |

**ポイント**:
- 「想定質問」列にユーザーが使いそうな言い回しを複数記載（スラッシュ区切り）
- 重要度列で `high` / `medium` / `low` を設定

### Step 2: スプレッドシート→JSONエクスポート

```json
{
  "question": "参加年齢は？",
  "similar_questions": [
    "何歳から参加できますか",
    "30歳未満でも参加できますか",
    "年齢制限はありますか"
  ],
  "answer": "WMG2027への参加資格は30歳以上（2027年12月31日時点）です。",
  "category": "参加資格",
  "priority": "high"
}
```

### Step 3: Gemini File Search Storeへの登録

1. JSONファイルをFile Search Storeにアップロード
2. `data/file-store-manifest.json` を更新
3. 既存のQ&Aデータと統合

### Step 4: 検証テスト

想定質問でチャットボットをテストし、正しいファイルが検索されるか確認:
- 「何歳から参加できますか？」→ 参加年齢のQ&Aがヒットするか
- 「登録締切はいつ？」→ 日程情報がヒットするか

## 補足: なぜセマンティック検索で「想定質問」が有効か

```
ユーザー質問: "何歳から参加できますか？"
            ↓ ベクトル化
       [0.23, 0.87, 0.12, ...]
            ↓ コサイン類似度計算

文書A（想定質問あり）: "何歳から参加できますか？" → 類似度: 0.95 ✅
文書B（回答のみ）: "30歳以上が参加資格です" → 類似度: 0.72
```

想定質問を含めることで、ユーザーの質問文との類似度が大幅に向上します。
