# rough-estimate スキル更新計画

## 概要
作成済み見積書（estimate-ec-bpaas.md）から学んだベストプラクティスを rough-estimate スキルに反映する。

## 変更対象ファイル

| # | ファイル | 変更種別 |
|---|----------|----------|
| 1 | `SKILL.md` | 編集 |
| 2 | `assets/template.md` | 編集 |
| 3 | `references/cost-estimation.md` | 編集 |
| 4 | `references/document-structure.md` | 編集 |
| 5 | `references/review-checklist.md` | **新規作成** |

---

## 1. SKILL.md の変更

### 1.1 エグゼクティブサマリーガイドライン追加
- 1ページで全体像把握できる構成
- 必須テーブル: プロジェクト概要 / スケジュール&費用
- 期待効果は箇条書き3点以内

### 1.2 クライアント向け情報開示ルール追加
削除すべき内部情報:
- 原価単価、販売単価、利益率
- バッファ（%、h）
- 工数詳細（h）→ フェーズ単位の金額のみ記載

### 1.3 AI関連タスクの工数見積りガイドライン追加
- LLM連携開発の工数係数（プロンプト設計: ×1.5-2.0）
- AI案件固有のリスクバッファ（8-12%）
- 早期検証タスク（Week 1必須）

---

## 2. template.md の変更

### 2.1 エグゼクティブサマリー強化
```markdown
| 項目 | 内容 |
|------|------|
| **対象** | {{1文で説明}} |
| **コア機能** | {{中心機能 + 価値}} |
| **差別化** | {{競合との違い}} |
```

### 2.2 技術スタックセクション追加
- レイヤー別テーブル（選定理由列付き）
- フォールバック案の明記

### 2.3 システム構成図の5層フロー標準化
```
データソース → データ基盤 → 可視化 → 意思決定AI → 施策実行
```

### 2.4 ROI試算セクション強化
- 前提条件の明記
- 計算式の可視化
- ※注記で補足説明

### 2.5 スコープ透明化
- 「含まれるもの」「含まれないもの」の明確化
- 変動要因（増加/減少）テーブル

## 3. cost-estimation.md の変更

### バッファ設定ガイドライン拡張

| 案件タイプ | 推奨バッファ |
|-----------|-------------|
| 通常開発 | 4-5% |
| AI/LLM連携 | 8-12% |
| 外部API依存 | +5% |
| POC案件 | 10-15% |

## 4. document-structure.md の変更

### 4.1 5層フローパターン追加
- Layer 1: データソース
- Layer 2: データ基盤
- Layer 3: 可視化
- Layer 4: 意思決定
- Layer 5: 実行

### 4.2 Ganttマイルストーン併記パターン
- `★マイルストーン` セクションを最上部に配置
- 詳細タスクは各Phaseセクションに分類

---

## 5. review-checklist.md（新規作成）

### チェック項目
1. **整合性チェック**: スコープ/日付/数値
2. **情報開示チェック**: 内部情報が含まれていないか
3. **計算根拠チェック**: ROI試算の透明性
4. **リスク評価チェック**: AI案件のバッファ確認
5. **構造チェック**: セクション番号、Mermaid構文
